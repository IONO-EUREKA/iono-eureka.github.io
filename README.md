### Content Workflows

#### Adding an event
Create a new file in `_events/` following the naming convention `YYYY-MM-DD-slug.md`:
```yaml
---
title: "Event Title"
date: 2026-10-01
display_date: "1â€“3 October 2026"

```

#### Adding a news post
Create a new file in `_news/` following `YYYY-MM-DD-slug.md`:
```yaml
---
title: "News Title"
date: 2026-03-18
summary: "One-sentence summary shown on the news listing page."
---
Full post content here.
```

#### Adding a member institution
Edit `_data/members.yml` â€” each entry needs: `id`, `name`, `city`, `country`, `lat`, `lon`.

### Publications

Publications are displayed on the Resources page and managed via `_data/publications.yml`.

**The recommended workflow is to export from NASA ADS and convert automatically:**

1. Open your ADS library at https://ui.adsabs.harvard.edu/user/libraries
2. Select all papers â†’ **Export â†’ BibTeX** â†’ save as e.g. `_bibliography/export-bibtex.bib`
3. Run the conversion script from the repo root:
   ```
   python scripts/ads_bib_to_yml.py _bibliography/export-bibtex.bib
   ```
4. Commit the updated `_data/publications.yml`

> **Note:** `_bibliography/` is listed in `.gitignore` â€” the raw `.bib` export is never committed. Only the generated `_data/publications.yml` is tracked.

The YAML file supports these fields per entry: `year`, `title`, `doi`, `authors`, `journal`, `volume`, `issue`, `pages`, `abstract`. Only `year`, `title`, and `doi` are currently displayed; adding more fields to the template is a one-line change.

### Repository Feed (GitHub Actions)

`.github/workflows/update-repos.yml` runs daily at 03:17 UTC and on manual trigger.
It calls `scripts/fetch_repos.py` to refresh `_data/repos_live.json` with live GitHub data.
No secrets need to be configured â€” it uses the built-in `GITHUB_TOKEN`.
